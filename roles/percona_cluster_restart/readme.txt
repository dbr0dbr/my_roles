Запуск:
На одной ноде (включенной первой) должен быть зпущен сервис mysql@bootstrap, на остальных запускаем просто mysql. 
После запуска mysql@bootstrap можно потушить и стартануть mysql, кластер уже запущен и будет работать.
Перезагружать ноды по одной можно, это не вызовет распада кластера.

Особенности:
На каждой ноде есть файл /var/lib/mysql/grastate.dat, в котором есть параметр "safe_to_bootstrap: ", если там стоит 0, то нода не готова стать главной, если 1, то готова. 
Если там у нас установленна 1, то мы можем сделать ноду главной выполнив systemctl stop mysql; systemctl start mysql@bootstrap 
После некорректного отключения всего кластера там окажется 0 на всех нодах, для того, чтобы кластер запустился, нужно на одной из нод вытавить "safe_to_bootstrap: 1" вручную.
Если работа кластера была завершена корректно, то на последней выключенной ноде "safe_to_bootstrap: " в /var/lib/mysql/grastate.dat будет установлен в 1.
Кластер сам не поднимится: на этой ноде нужно руками выполнить systemctl start mysql@bootstrap, после чего запустить mysql на сотальных нодах.
